# Правила Валидации BrainMessenger (Система Целостности Данных)

### 1. Введение: Валидация как Первая Линия Защиты

**Описание:** Этот документ определяет все правила валидации для данных, вводимых пользователем в BrainMessenger. Валидация — это критически важный компонент системы, который обеспечивает целостность и безопасность данных (Принцип 7), а также предоставляет ясную обратную связь (Принцип 14) пользователю.

**Принципы:**
*   **Двухуровневая Валидация:** Валидация ВСЕГДА происходит на клиенте (Frontend) и на сервере (Backend). Backend-валидация является "единым источником правды".
*   **Ясность Ошибок:** Сообщения об ошибках должны быть понятны пользователю и подсказывать, как исправить проблему.

### 2. Правила для Потока Регистрации (`/register`)

#### **Шаг 1: Создание Пароля**

*   **Поле:** `password`
    *   **Правило 1:** Обязательное поле (`required`).
        *   *Сообщение об ошибке (ключ i18n):* `validation.password.required` ("Пароль обязателен")
    *   **Правило 2:** Минимальная длина — 8 символов.
        *   *Сообщение об ошибке:* `validation.password.minLength` ("Пароль должен содержать минимум 8 символов")
    *   **Правило 3:** Должен содержать как минимум одну заглавную букву (`A-Z`).
        *   *Сообщение об ошибке:* `validation.password.uppercase` ("Должен содержать заглавную букву")
    *   **Правило 4:** Должен содержать как минимум одну цифру (`0-9`).
        *   *Сообщение об ошибке:* `validation.password.digit` ("Должен содержать цифру")
    *   **Правило 5:** Должен содержать как минимум один спецсимвол (`!@#$%^&*`).
        *   *Сообщение об ошибке:* `validation.password.specialChar` ("Должен содержать спецсимвол")
    *   **Примечание:** Поле `confirmPassword` отсутствует для упрощения и ускорения процесса регистрации.

#### **Шаг 2: Ввод Имени**

*   **Поле:** `name`
    *   **Правило 1:** Обязательное поле (`required`).
        *   *Сообщение об ошибке:* `validation.name.required` ("Имя обязательно")
    *   **Правило 2:** Минимальная длина — 2 символа.
        *   *Сообщение об ошибке:* `validation.name.minLength` ("Имя должно быть длиннее 2 символов")
    *   **Правило 3:** Максимальная длина — 50 символов.
        *   *Сообщение об ошибке:* `validation.name.maxLength` ("Имя должно быть короче 50 символов")

#### **Шаг 4: Ввод Email**

*   **Поле:** `email`
    *   **Правило 1:** Обязательное поле (`required`).
        *   *Сообщение об ошибке:* `validation.email.required` ("Email обязателен")
    *   **Правило 2:** Должен быть валидным email-адресом.
        *   *Сообщение об ошибке:* `validation.email.invalid` ("Введите корректный email")
    *   **Правило 3:** **Email должен быть от провайдера Gmail.** Адрес должен заканчиваться на `@gmail.com`.
        *   *Сообщение об ошибке:* `validation.email.gmailOnly` ("Поддерживается регистрация только с Gmail-аккаунтов")
    *   **Правило 4 (Только на Backend):** Email должен быть уникальным в базе данных.
        *   *Сообщение об ошибке:* `validation.email.inUse` ("Этот email уже используется")

#### **Шаг 5: Верификация Кода**

*   **Поле:** `verificationCode`
    *   **Правило 1:** Длина — **8 символов**.
    *   **Правило 2:** Может содержать **любые** символы (буквы, цифры, спецсимволы).
    *   **Правило 3 (Только на Backend):** Должен совпадать с кодом, сгенерированным для данного email.
        *   *Сообщение об ошибке:* `validation.code.invalid` ("Неверный код подтверждения")

### 3. Техническая Реализация

*   **Frontend (Next.js):** Валидация реализуется с помощью `react-hook-form` и/или `Zod`. Правило для проверки `@gmail.com` можно реализовать с помощью регулярного выражения.
*   **Backend (NestJS):** Валидация реализуется с помощью `class-validator` и `DTO`. Для проверки `@gmail.com` также используется декоратор `@Matches(/@gmail\.com$/)`.
