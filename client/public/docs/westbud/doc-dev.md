# Руководство по Разработке (Developer Guide)

### 1. Философия Кода
Мы используем принцип **KISS (Keep It Simple, Stupid)**. Сайт должен быть легко поддерживаемым даже другим разработчиком.

### 2. Структура Проекта
```text
/src
  /assets      # Исходные изображения (не оптимизированные)
  /components  # Переиспользуемые UI-блоки (Button, Card, Section)
  /layout      # Header, Footer
  /styles      # Глобальные стили и Tailwind Config
  /utils       # Хелперы (валидация, форматирование)

3. Стандарты Именования
Файлы: PascalCase для компонентов (ProjectCard.tsx), camelCase для утилит (formatDate.ts).
CSS Классы: Используем утилиты Tailwind. Если нужен кастомный CSS, используем методологию BEM в index.css.
4. Работа с Изображениями
Все фото должны быть конвертированы в .webp перед деплоем.
Для тяжелых изображений (Hero Section) обязательно использовать атрибут priority (если Next.js) или fetchpriority="high".

### **Файл 6: `docs/doc-validation.md` (Протокол Валидации)**

Описывает, как мы защищаем "входную дверь" (форму) от мусора.

```markdown
# Протокол Валидации Данных

### 1. Введение
Валидация формы — критический этап для фильтрации спама и получения качественных данных для отдела продаж.

### 2. Правила Валидации (Zod / HTML5)

#### Поле: "Full Name"
*   **Правило:** Обязательное. Минимум 2 символа.
*   **Ошибка:** "Please enter your name."

#### Поле: "Phone Number"
*   **Правило:** Обязательное. Должно соответствовать формату (Regex): `^[+]?[(]?[0-9]{3}[)]?[-\s.]?[0-9]{3}[-\s.]?[0-9]{4,6}$`.
*   **Ошибка:** "Invalid phone number format."

#### Поле: "Email"
*   **Правило:** Обязательное. Стандартная проверка email-адреса.
*   **Ошибка:** "Please enter a valid email address."

### 3. UX Обработки Ошибок
*   Ошибки должны появляться **под полем** ввода красным цветом (`text-red-500`).
*   Кнопка "Send Request" блокируется (`disabled`), пока форма не валидна.
