# Техническая Документация Westbud Redesign

### 1. Философия Стека: Прагматизм и Производительность

Выбор технологий для этого проекта был продиктован **Принципом 12 (Прагматизм и Реализм)**. Мы использовали инструменты, которые обеспечивают максимальную скорость загрузки для пользователя (Google Ads performance) и удобство поддержки для разработчика.

### 2. Core (Ядро Системы)

*   **[React 19](https://react.dev/)**: Библиотека для построения пользовательского интерфейса. Выбрана за компонентный подход, позволяющий легко переиспользовать элементы (кнопки, карточки) и поддерживать чистоту кода.
*   **[TypeScript](https://www.typescriptlang.org/)**: Язык разработки. Обеспечивает **строгую типизацию**, что предотвращает 90% ошибок "в рантайме" и делает код предсказуемым и безопасным (Safety First).
*   **[Vite](https://vitejs.dev/)**: Сборщик нового поколения. Обеспечивает мгновенный запуск dev-сервера и оптимизированную сборку для продакшена.

### 3. Стилизация и Дизайн

*   **[Tailwind CSS](https://tailwindcss.com/)**: Утилитарный CSS-фреймворк. Позволил создать уникальную дизайн-систему (цвета BESIX, отступы) без написания громоздких CSS-файлов. Гарантирует минимальный размер итогового CSS-бандла.
*   **Framer Motion / CSS Keyframes**: Использованы для создания плавных анимаций появления (Scrollytelling), которые повышают вовлеченность пользователя, но не перегружают браузер.

### 4. Архитектура и Логика

*   **[React Router v6](https://reactrouter.com/)**: Обеспечивает клиентский роутинг (SPA). Переходы между страницами происходят мгновенно, без перезагрузки сервера.
*   **[i18next](https://www.i18next.com/)**: Промышленный стандарт для интернационализации. Все тексты вынесены в отдельные JSON-файлы, что позволяет легко добавлять новые языки в будущем без изменения кода.
*   **Modular Architecture**: Данные проектов (`projects.ts`) отделены от логики отображения. Это позволяет клиенту добавлять новые проекты, просто редактируя один файл с данными, не касаясь верстки.

### 5. Интеграция (Backend)

*   **Build Adaptation**: Конфигурация Vite настроена специально для интеграции с **Python/Flask**.
    *   Скрипт `build:flask` меняет базовые пути ассетов, чтобы Flask корректно отдавал статику из папки `/static`.
*   **Forms**: Форма контактов настроена на отправку данных на эндпоинт `/send-message`, что позволяет легко подключить любую логику отправки email на стороне Python.

### 6. Оптимизация (Performance)

*   **Lazy Loading**: Тяжелые компоненты и изображения оптимизированы для быстрой первой отрисовки (FCP).
*   **Code Splitting**: Vite автоматически разделяет код на чанки для оптимальной загрузки.
