# Руководство по Производительности (Performance)

### 1. Целевые Показатели (KPI)
*   **LCP (Largest Contentful Paint):** < 2.5 сек.
*   **CLS (Cumulative Layout Shift):** < 0.1 (Никаких прыгающих блоков).
*   **FID (First Input Delay):** < 100 мс.

### 2. Стратегия Оптимизации

#### 2.1. Изображения (Lazy Loading)
*   Изображения "ниже сгиба" (Below the fold) должны иметь атрибут `loading="lazy"`.
*   Используются современные форматы: AVIF или WebP.

#### 2.2. Шрифты (Font Optimization)
*   Шрифт `Sora` подключается локально или через Google Fonts с параметром `display=swap`, чтобы текст был виден сразу, даже до загрузки шрифта.

#### 2.3. Скрипты
*   Google Analytics и Pixel должны загружаться в режиме `defer` или через Google Tag Manager, чтобы не блокировать основной поток рендеринга.

### 3. Минификация
*   В продакшн сборке (Vite build) удаляются все `console.log` и комментарии. CSS и JS минифицируются.
