# Спецификация Требований Westbud

### 1. Введение
Этот документ описывает требования к корпоративному сайту строительной компании Westbud. Цель системы — конвертация трафика (Google Ads) в целевые лиды.

### 2. Функциональные Требования (FR)

#### FR-1: Навигация и Структура
*   **FR-1.1:** Хедер должен быть "липким" (Sticky) и содержать навигацию: Home, Projects, Services, Contact.
*   **FR-1.2:** В мобильной версии должно использоваться меню-"гамбургер" с плавной анимацией открытия.

#### FR-2: Презентация Проектов (Портфолио)
*   **FR-2.1:** Секция "Projects" должна быть реализована в виде сетки (Grid).
*   **FR-2.2:** Карточка проекта должна содержать фото, название, локацию и статус (Завершен/В процессе).
*   **FR-2.3:** При наведении на карточку должен срабатывать микро-зум изображения (Visual Feedback).

#### FR-3: Лидогенерация (Форма)
*   **FR-3.1:** Форма контактов должна содержать поля: Имя, Телефон, Email, Сообщение.
*   **FR-3.2:** После успешной отправки должно появляться модальное окно или Toast-уведомление ("Спасибо, мы свяжемся").

#### FR-4: Мультиязычность
*   **FR-4.1:** Поддержка переключения языков (EN / NL / UA - опционально).

### 3. Нефункциональные Требования (NFR)

*   **NFR-1 (SEO):** Семантическая верстка (H1-H6), наличие `alt` у всех изображений, Open Graph теги для соцсетей.
*   **NFR-2 (Performance):** Google Lighthouse Score > 90 (Green Zone) для мобильных устройств.
*   **NFR-3 (Accessibility):** Контрастность текста должна соответствовать стандарту WCAG AA (цвета BESIX это позволяют).
