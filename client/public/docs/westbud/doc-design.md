# Westbud Design System: Color Palette

## 1. Философия Цвета: "Инженерная Эстетика"

Наша цветовая система построена на принципах **надежности, ясности и функциональности**. Мы избегаем визуального шума, используя цвета для создания иерархии и управления вниманием пользователя.

Это не просто цвета, это **сигналы**:
*   **Синий (#1B2A41)** — это "Сталь и Чертежи". Фундамент, стабильность, корпоративная надежность.
*   **Оранжевый (#F59E0B)** — это "Безопасность и Действие". Цвет строительных касок, предупреждающих знаков и активных действий.
*   **Серый и Белый** — это "Бетон и Пространство". Чистый холст для контента.

## 2. Палитра Токенов (Token Reference)

Мы используем семантическое именование токенов. Это позволяет менять конкретные оттенки в будущем, не ломая логику системы.

| Token | HEX Code | Visual | Semantic Role (Роль) |
| :--- | :--- | :--- | :--- |
| **Primary** | `#1B2A41` | 🔵 | **Бренд и Структура.** Основной цвет навигации, заголовков и футера. Символ компании. |
| **Accent** | `#F59E0B` | 🟠 | **Действие (CTA).** Кнопки, иконки преимуществ, подсветка важных цифр. "Нажми меня" или "Посмотри сюда". |
| **Background**| `#F3F4F6` | 🌫️ | **Среда (Бетон).** Фон секций. Создает контраст с белыми карточками. (Tailwind: `gray-100`) |
| **Text** | `#1F2937` | ⚫ | **Контент.** Основной цвет текста для максимальной читаемости. Не чистый черный, чтобы снизить нагрузку на глаза. (Tailwind: `gray-800`) |
| **Base** | `#FFFFFF` | ⚪ | **Поверхность.** Фон карточек проектов, модальных окон и основного холста. |

## 3. Техническая Реализация (Tailwind Config)

Чтобы использовать эту систему в коде, добавьте следующие настройки в `tailwind.config.js`. Это превращает цвета в **утилитарные классы** (например, `bg-primary`, `text-accent`).

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: [
    "./index.html",
    "./src/**/*.{js,ts,jsx,tsx}",
  ],
  theme: {
    extend: {
      colors: {
        primary: {
          DEFAULT: '#1B2A41', // Deep Navy
          light: '#2C3E50',   // Optional: lighter shade for hover
        },
        accent: {
          DEFAULT: '#F59E0B', // Construction Amber
          hover: '#D97706',   // Darker amber for hover states
        },
        background: '#F3F4F6', // Light Concrete
        text: '#1F2937',       // Dark Charcoal
        base: '#FFFFFF',       // Pure White
      },
      fontFamily: {
        sans: ['Sora', 'sans-serif'], // Наш основной шрифт
      }
    },
  },
  plugins: [],
}
```

## 4. Правила Использования (Dos and Don'ts)

### ✅ DO (Как делать правильно)
*   **Иерархия**: Используйте **Primary** для крупных блоков (Header, Footer) и заголовков секций.
*   **Акцент**: Используйте **Accent** точечно. Одна-две кнопки на экран, иконки в списке преимуществ. Оранжевый должен "гореть" на фоне синего или белого.
*   **Глубина**: Используйте **Background** (`#F3F4F6`) для всей секции, а поверх него кладите карточки с цветом **Base** (`#FFFFFF`) и легкой тенью (`shadow-md`). Это создает эффект "объема".

### ❌ DON'T (Как делать нельзя)
*   Не используйте **Accent** для текста: Оранжевый текст на белом фоне может быть трудночитаемым (плохой контраст). Используйте его для графики или фона кнопок с белым или темным текстом.
*   Не красьте всё в **Primary**: Если весь сайт будет темно-синим, он станет мрачным. Балансируйте "тяжелый" синий большим количеством "воздуха" (Base/Background).
*   Не используйте чистый черный (`#000000`): Для текста используйте наш токен **Text** (`#1F2937`). Это выглядит дороже и профессиональнее.

### **Почему это важно?**
Эта документация — ваш **"единый источник правды"**. Когда вы (или кто-то другой) будете верстать новую страницу через полгода, вам не нужно будет пипеткой искать цвета. Вы просто откроете этот файл и вспомните: "Ага, кнопки у нас `bg-accent`, а фон секций `bg-background`". Это **система**, которая экономит время.
